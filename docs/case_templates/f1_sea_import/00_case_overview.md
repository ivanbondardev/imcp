# F1 Sea Import — Case Overview

**Case Type:** `F1_SEA_IMPORT`  
**Domain:** Logistics / Sea Freight  
**Route:** China (Yiwu/Shenzhen warehouse) → Ukraine (Kyiv terminal + unloading point)  
**Primary Operator:** Logistics Manager (Sales/Operations side)  
**IMCP Compatibility:** Core v1.1

## 1) Business Goal
Швидко та безпомилково провести кейс морського імпорту з Китаю:
- отримати повні вхідні дані від клієнта;
- сформувати та погодити комерційну пропозицію (quote);
- створити запит/угоду в 1С;
- забезпечити правильні габарити для коносамента та пакувального листа;
- організувати взаємодію з ЗЕД, брокером, складом;
- провести документальну частину без розбіжностей.

---

## 2) Definition of Done (ідеально завершений кейс)
Кейс вважається завершеним успішно, якщо:
- клієнт підтвердив перевезення;
- угода "імпорт" створена в 1С;
- дані по вантажу (кількість місць, габарити, штабелювання, опис) підтверджені складом;
- коносамент (BL) погоджений без розбіжностей з контрактом та інвойсом;
- довіреність на експедирування оформлена та надіслана в оригіналі;
- брокер підготував ПП/ПД та ЗДП;
- в кейсі збережено повну історію (хто що підтвердив, коли, на основі яких даних).

---

## 3) Start / End Boundaries

### Start (коли кейс стартує)
- надходить запит від клієнта на морське перевезення з Китаю

### End (коли кейс завершується в межах цього шаблону)
- документи та доручення передані, BL погоджено, інструкції на оформлення готові

### Next Template (що далі)
Цей шаблон охоплює **документальну підготовку**. Після `state = READY_FOR_CLEARANCE` або `CLOSED`:
- **F1_SEA_IMPORT_DELIVERY** (опціонально) — фізична доставка, трекінг, arrival confirmation
- **F1_SEA_IMPORT_CLEARANCE** (опціонально) — митне оформлення, закриття угоди в 1С

---

## 4) Participants (Roles)
- **Client** — надає дані по вантажу, підтверджує перевезення, оформлює довіреність
- **Logistics Manager** — веде кейс, комунікує, рахує quote, затверджує критичні рішення
- **China Warehouse (Yiwu/Shenzhen)** — підтверджує фактичні габарити/місця
- **ZED Department** — готує/передає драфт BL, супроводжує зовнішньоекономічні документи
- **Broker** — перевіряє документи, визначає потребу експортної декларації, готує ПП/ПД та ЗДП
- **Logistics Delivery Team (UA)** — запитує інструкції/довіреність за 3 тижні до прибуття

## 5) Core Artifacts
- **Client Intake Form** (7 обов'язкових питань + 3 safety/context поля)
- **1C Request**
- **Quote (pricing) + validity note "7–9 days"**
- **1C Import Deal**
- **Warehouse Instructions + Marking "ALEX-<deal_number>"**

- **Packing List aligned with warehouse dims**
- **POA / Довіреність на експедирування**
- **Broker docs: ПП/ПД, ЗДП**

## 6) Main Risks (Top 10)
1. Неповні дані від клієнта → неправильний quote
2. Розбіжність заявлених габаритів vs фактичних (склад) → проблеми з BL та ціною
3. Небезпечний вантаж без маркування/опису → затримки/ризики
4. Невизначений брокер (наш/клієнта) → стоп процесу
5. Експортна декларація потрібна, але не враховано → VAT/ПДВ ризики
6. Немає телекс-релізу → затримки видачі
7. BL draft має помилки (seller/buyer) → юридичні/митні ризики
8. Не узгоджена страховка → фінансові ризики
9. Не підготовлена довіреність вчасно → зрив оформлення
10. Комунікаційні розриви між ЗЕД/брокером/менеджером → втрати часу

---

## 7) Human-in-the-Loop Focus
Ключові рішення, які завжди підтверджує людина:
- фінальна ціна/умови quote
- створення угоди "імпорт" в 1С
- підтвердження зміни габаритів (якщо відрізняються)
- рішення про страхування
- підтвердження BL draft
- підтвердження готовності довіреності (скан + оригінал)
